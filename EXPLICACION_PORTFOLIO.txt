EXPLICACION DEL PROYECTO PORTFOLIO2
==================================

Este documento resume los archivos principales del proyecto, que hace cada uno
y que se ha cambiado en ellos para llevar la web desde una base inspirada en
montone.studio hacia una version mas personal para jgarcia3199.


1. ESTRUCTURA GENERAL
=====================

El proyecto esta hecho con Next.js usando App Router.

Archivos base:
- app/layout.tsx
- app/page.tsx
- app/globals.css

Componentes principales:
- app/components/Navbar.tsx
- app/components/Cursor.tsx
- app/components/Hero.tsx
- app/components/Statement.tsx
- app/components/Services.tsx
- app/components/Portfolio.tsx
- app/components/Testimonials.tsx
- app/components/About.tsx
- app/components/Footer.tsx


2. ARCHIVOS BASE
================

2.1 app/layout.tsx
------------------
Que hace:
- Define la estructura HTML global de toda la app.
- Carga las fuentes Geist y Geist Mono.
- Importa los estilos globales.
- Define el metadata principal de la pagina.

Que se cambio:
- Se actualizo el metadata para que el titulo y la descripcion reflejen el
  portfolio de jgarcia3199.
- Se elimino el overflow-hidden del body. Antes bloqueaba o complicaba el
  scroll general de la pagina, y esta web depende mucho del scroll.
- Ahora lee cookies de idioma y paleta para pintar desde servidor:
  - atributo lang correcto
  - data-theme correcto
  Esto evita que el HTML inicial salga con una configuracion distinta a la
  que luego usa el navegador al hidratar.


2.2 app/page.tsx
----------------
Que hace:
- Es la pagina principal.
- Monta todos los componentes en orden.
- Inicializa Lenis para el smooth scroll.
- Activa las animaciones fade-up con IntersectionObserver.

Que se cambio:
- Ahora es un Server Component muy fino.
- Lee cookies de preferencia:
  - idioma
  - paleta
- Pasa esos valores iniciales ya resueltos a HomeClient.
- Esta separacion evita hydration mismatch porque el servidor y el cliente
  parten de la misma informacion.


2.3 app/globals.css
-------------------
Que hace:
- Contiene casi toda la identidad visual del proyecto.
- Define variables CSS, fondos, tipografia, espaciados, animaciones y estilos
  de cada bloque de la home.
- Controla navbar, overlay menu, hero, statement, cards, carruseles, footer,
  responsive y sistema de paletas.

Que se cambio:
- Se reescribio casi por completo.
- Se creo una identidad visual inspirada en la referencia pero mas propia.
- Se implemento el sistema de variables de color para temas.
- Se dejaron finalmente dos temas activos:
  1. neo-mint
  2. lapreviapp
- Se adapto el glow del cursor para reaccionar al tema.
- Se ajustaron las transiciones entre secciones para dar mas continuidad.
- Se corrigio el layout de las cards de servicios:
  - mas separacion entre numero y titulo
  - grid mas amplia
  - offset mas coherente
- Se arreglo la zona del carrusel horizontal de proyectos:
  - viewport real
  - cards con preview embebido
- Se adapto la zona de stack/herramientas:
  - carrusel vertical mas controlable
- Se mejoro el selector visual de paletas en el menu.

13. HYDRATION MISMATCH
======================

Se ha corregido un hydration mismatch en la navbar.

Causa:
- el servidor renderizaba una paleta e idioma por defecto
- el cliente podia leer otros valores desde localStorage al hidratar
- React detectaba diferencias en la clase activa del selector de paletas
- ademas, app/page.tsx hacia un branch con typeof window para el idioma

Solucion:
- Se creo app/lib/preferences.ts para centralizar:
  - claves de cookies y localStorage
  - valores por defecto
  - parseo seguro de idioma y paleta
- app/layout.tsx ahora lee cookies y pinta lang y data-theme desde servidor
- app/page.tsx lee cookies y pasa initialLocale e initialTheme a HomeClient
- HomeClient y Navbar guardan los cambios en cookie y localStorage
- asi servidor y cliente arrancan ya alineados y desaparece el mismatch real

Nota:
- El atributo cz-shortcut-listen que podia aparecer en body viene de una
  extension del navegador, no del codigo del proyecto.
